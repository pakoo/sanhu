## v1.0 — 基础持仓管理工具 ✅

**定位**：计算器 + 可视化，告诉你「现状是什么」

**已实现**：
- 持仓总览仪表盘（总资产、收益、资产配置饼图）
- 风险分析（风险评分、预警、相关性热力图、波动率）
- 调仓建议（当前 vs 目标配置对比、渐进式调仓计划）
- 策略回测（定投回测、止盈定投回测、收益曲线可视化）
- 数据来源：天天基金 Eastmoney 公开 API，本地 SQLite 存储

**已知局限**：
- 纯规则引擎，无市场估值判断
- 不感知基金内部持仓（不知道买了什么股票）
- 数据需手动刷新
- 没有发现新基金的能力

## v1.5 — 基金持仓透视 🔜

**定位**：穿透你真正买了什么，解决「持有8只基金却不知道里面装着什么」的核心痛点

**范围说明**：
- 权益类基金（股票型、混合型、QDII）→ 做完整持仓穿透
- 债券型基金 → 只展示资产配比（股/债/现金），不拆解债券持仓

---

### 功能一：单基金持仓详情页

入口：点击持仓表格中的基金名称（扩展现有弹窗）

展示内容：
- 前十大持仓股票：名称、占基金净值比例、较上期变化（↑↓）
- 行业分布饼图：科技/消费/金融/医药/新能源/其他
- 资产配比：股票% / 债券% / 现金%
- 数据时效标注：「截至 XXXX 年 X 季度」（公募基金季报，每季度披露一次）

数据来源：
```
持仓股票：https://fundf10.eastmoney.com/FundArchivesDatas.aspx?type=jjcc&code={code}&topline=10
行业分布：https://fundf10.eastmoney.com/FundArchivesDatas.aspx?type=hycc&code={code}
```

---

### 功能二：跨基金持仓重叠分析（核心亮点）

入口：新增「持仓透视」Tab

**重叠股票列表**
- 合并所有权益类基金的前十大持仓
- 标出在多只基金中重复出现的股票
- 计算实质暴露比例（加权到总资产维度）
- 示例：宁德时代同时出现在国金量化(3%)、永赢制造(5%)、东方低碳(4%)，三只基金合计占总资产 X%，实质暴露 = X% × 各自占比之和

**真实分散度评分**
- 基于 Herfindahl 指数计算持仓集中度
- 输出：「你持有 5 只权益基金，实质分散度约等于 2.3 只独立基金」

**穿透后行业分布**
- 跨越所有权益基金的汇总行业饼图
- 每只基金的贡献用颜色分层显示（堆叠条形图）

---

### 功能三：持仓变化追踪

逻辑：存储每次抓取的季报持仓，对比相邻两期

展示：
- 新进持仓（本期有、上期没有）→ 绿色
- 退出持仓（上期有、本期没有）→ 红色
- 大幅加/减仓（变化 > 1%）→ 带方向箭头

---

### 同步：定时自动刷新净值

方案：APScheduler 后台任务，应用启动时注册

```
每日 15:30（周一至周五） → 自动执行 refresh_all_funds()
每日 16:00（周一至周五） → 自动抓取最新季报持仓（如有更新）
```

页面顶部显示「最后更新时间」，无需手动点刷新。

## v2.0 — 智能决策层 📋

**定位**：从「告诉你现状」升级到「给你可执行的建议」

### 估值信号（PE百分位）

- 接入沪深300、中证500的 PE 历史数据
- 计算近10年百分位：< 30% 低估 / 30-70% 合理 / > 70% 高估
- 在调仓建议 Tab 加估值参考条，高估时建议权益配置下调 5%，低估时上调 5%

### 基金综合评分

5个维度合成 0-100 分（基于近1年 nav_history 数据）：
- 夏普比率 30%
- 最大回撤 25%
- 年化收益率 25%
- 波动率 20%

持仓表格加「评分」列，调仓推荐的基金按分数排序。

### Claude AI 每日持仓解读

- 每日收盘后自动触发，调用 Claude API
- 输入：当日主要指数涨跌、持仓各基金涨跌、当前配置结构、已触发预警
- 输出：200字以内的持仓解读 + 短期关注点
- 展示：总览 Tab 顶部「今日解读」卡片
- 需配置：`ANTHROPIC_API_KEY`

## v2.5 — 投资标的挖掘 💡

**定位**：覆盖投资决策链路的前半段——发现值得投资的新基金

**已明确的约束**：
- 仅限支付宝可购买的基金
- 被动推荐（系统根据持仓缺口主动推荐，而非用户主动搜索）

**待设计的核心问题**：
- 推荐策略：如何定义「适合当前持仓缺口」的基金
- 筛选维度：业绩、基金经理、规模费率、与现有持仓的相关性如何加权
- 支付宝可购清单的获取方式

> 此模块策略待 v2.0 完成后，与用户共同设计。

## 决策模型演进路径

```
v1.0  规则引擎          → 「你的债基占85%，偏多」
v1.5  + 持仓穿透        → 「你实质上重仓宁德时代，跨3只基金」
v2.0  + 市场信号 + AI   → 「当前市场低估，建议小幅加仓，重点关注宁德时代集中风险」
v2.5  + 标的发现        → 「基于你的缺口，以下3只基金值得考虑」
```
